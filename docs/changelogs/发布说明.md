# Word文档格式修复工具 - 更新日志

## v1.1.2 (2026-01-13)

### 修复内容

1. **UI边框问题修复**
   - 消除了鼠标聚焦到窗口内部模块时显示的黑色线条
   - 移除了Canvas、Listbox、Text等控件的焦点边框
   - 优化了输入控件的边框样式，确保视觉一致性

2. **界面美化**
   - 为所有Frame组件添加了统一的样式配置
   - 调整了控件的边框宽度和样式，提升整体美观度
   - 确保控件在聚焦时不会显示突兀的黑色边框

### 技术变更

- 为TFrame添加了样式配置，设置borderwidth=0和relief='flat'
- 为Canvas控件添加了highlightthickness=0参数
- 优化了TEntry和TCombobox的样式设置
- 为Listbox和Text控件添加了highlightthickness=0参数
- 为所有相关控件设置了适当的边框宽度和样式

### 测试结果

- UI界面测试：通过
- 边框显示测试：通过
- 控件聚焦测试：通过
- 视觉一致性测试：通过

## v1.1.1 (2026-01-12)

### 修复内容

1. **布局优化**
   - 移除了固定宽度限制，内容现在可以充满整个窗口
   - 消除了两侧固定边距，彻底解决留白问题
   - 保持了内容的响应式调整能力

2. **视觉一致性**
   - 为画布设置了与内部框架一致的背景颜色
   - 解决了底部背景与内容区域颜色不一致的问题

### 技术变更

- 修改了 `_on_canvas_configure` 函数，移除了宽度限制和边距
- 将 `max_content_width` 设置为无限大，实现完全响应式布局
- 将 `padding` 设置为 0，消除所有固定边距
- 为 Canvas 组件添加了背景颜色设置

### 测试结果

- 性能测试：通过
- 配置默认值测试：通过
- 工具函数测试：全部通过
- UI初始化测试：通过
- 预设配置测试：因环境问题未执行（与代码修改无关）

### 使用建议

- 窗口现在可以充分利用整个屏幕空间
- 内容会自动适应不同大小的窗口
- 界面更加整洁，没有不必要的留白
